# Day 08 – Cloud Server Setup: Docker, Nginx & Web Deployment

## Deploy a Real Web Server on the Cloud

### Step 1: Launch an AWS EC2 Instance

Created an Ubuntu EC2 instance from the AWS Management Console.

---

### Step 2: Connect to the Instance Using SSH

**Command:**

```bash
ssh -i "kg-p.pem" ubuntu@ec2-13-207-40-107.ap-south-1.compute.amazonaws.com
```

Successfully connected to the EC2 instance using SSH.

---

### Step 3: Update the Server

**Commands:**

```bash
sudo apt update
sudo apt upgrade -y
```

Updated the package list and installed the latest available updates.

---

### Step 4: Install Docker

**Commands:**

```bash
sudo apt install docker.io -y
docker --version
sudo systemctl enable docker
sudo systemctl start docker
sudo systemctl status docker
```

Verified that the Docker service was installed and running successfully.

---

### Step 5: Install Nginx

**Commands:**

```bash
sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx
sudo systemctl status nginx
```

Installed and started the Nginx web server.

---

### Step 6: Configure Security Group

Added an **Inbound Rule** in the AWS EC2 Security Group.

* Type: HTTP
* Port: 80
* Source: Anywhere (0.0.0.0/0)

Verified the Nginx welcome page by opening the EC2 Public IP in the browser.

---

### Step 7: Check Nginx Logs

**Commands:**

```bash
journalctl -u nginx
```

Viewed the Nginx service logs managed by systemd.

```bash
sudo cat /var/log/nginx/access.log
```

Viewed the HTTP access logs generated by Nginx.

---

### Step 8: Save Logs to a File

**Commands:**

```bash
sudo cp /var/log/nginx/access.log ~/nginx-logs.txt
sudo chown ubuntu:ubuntu ~/nginx-logs.txt
```

Verified the copied log file:

```bash
cat ~/nginx-logs.txt
```

---

## Challenges Faced

### 1. SSH Connection Failed

**Issue:**

The private key file (`kg-p.pem`) had incorrect permissions.

**Solution:**

```bash
chmod 400 kg-p.pem
```

After changing the permissions, SSH authentication worked successfully.

---

### 2. Unable to Access the Nginx Webpage

**Issue:**

The Nginx welcome page was not accessible through the browser.

**Solution:**

Added an HTTP (Port 80) inbound rule in the AWS Security Group, allowing web traffic to reach the EC2 instance.

---

### 3. Permission Denied While Reading Log File

**Issue:**

The copied `nginx-logs.txt` file was owned by the root user, resulting in a "Permission denied" error.

**Solution:**

```bash
sudo chown ubuntu:ubuntu ~/nginx-logs.txt
```

Changed the ownership so the file could be accessed without using `sudo`.
