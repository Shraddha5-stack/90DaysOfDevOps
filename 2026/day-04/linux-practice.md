# Linux Practice – Processes and Services

## Objective

Today's practice focused on understanding how Linux manages processes, services, and system logs. I explored commonly used commands to inspect running processes, verify service status, and troubleshoot issues using logs.

---

# Process Checks

## 1. Display Running Processes

```bash
ps -ef | head
```

**Purpose:** Displays the first few running processes along with details such as the Process ID (PID), parent process, user, and command.

**Learning:** This command provides a quick overview of active system processes and is useful for identifying running applications.

---

## 2. Monitor System Resources

```bash
top
```

**Purpose:** Displays CPU usage, memory consumption, load average, and currently running processes in real time.

**Learning:** `top` is one of the most useful commands for monitoring overall system performance and identifying resource-intensive processes.

---

# Service Checks

## 3. List Active Services

```bash
systemctl list-units --type=service --state=running
```

**Purpose:** Lists all services that are currently active on the system.

**Learning:** This command helps verify which services are running and is useful during server health checks.

---

## 4. Check SSH Service Status

```bash
systemctl status ssh
```

**Purpose:** Displays the current status of the SSH service, including whether it is active, inactive, or failed.

**Learning:** Checking service status is one of the first steps when troubleshooting service-related issues.

---

# Log Checks

## 5. View SSH Service Logs

```bash
journalctl -u ssh --no-pager | tail -20
```

**Purpose:** Displays the latest log entries generated by the SSH service.

**Learning:** Service logs help identify login attempts, errors, and unexpected service behavior.

---

## 6. Review System Logs

```bash
tail -20 /var/log/syslog
```

**Purpose:** Shows the most recent entries from the system log file.

**Learning:** System logs provide valuable information for diagnosing application, service, and operating system issues.

# Mini Troubleshooting Activity

### Scenario

I wanted to verify whether the SSH service was running correctly and check if any recent errors had been recorded.

### Troubleshooting Steps

* Listed all active services using `systemctl`.
* Verified the SSH service status.
* Examined recent SSH logs using `journalctl`.
* Reviewed the latest system log entries for additional information.

### Outcome

The service status and log analysis confirmed the operational state of the service and demonstrated how Linux logs can assist in troubleshooting and root cause analysis.
